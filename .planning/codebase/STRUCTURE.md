## Directory Purposes

**`extension/`:**
- Purpose: Chrome extension for Gmail integration, email classification, tracking pixel injection
- Contains: Manifest, background service worker, content scripts, sidebar UI, email classifier
- Key files: `manifest.json`, `background.js`, `content.js`, `sidebar.js`, `classifier.js`

**`server/`:**
- Purpose: Express.js API server for CRUD, analytics, external service integration
- Contains: REST endpoints, Prisma models, email finder utility
- Key files: `index.js` (main server), `emailFinder.js`, `prisma/schema.prisma`

**`web/`:**
- Purpose: React dashboard for viewing/managing tracked outreach in kanban board
- Contains: React components, hooks, utilities, styling via Tailwind CSS
- Key files: `src/App.jsx`, `src/components/`, `src/hooks/useOutreach.js`

**`shared/`:**
- Purpose: Shared constants used across extension, server, and web
- Contains: Kanban column names, storage keys
- Key files: `constants.js`

**`.planning/codebase/`:**
- Purpose: Generated architecture and code analysis documents
- Contains: ARCHITECTURE.md, STRUCTURE.md, and other GSD analysis docs
- Generated: Yes (by orchestrator)
- Committed: Yes

## Key File Locations

**Entry Points:**

- `extension/manifest.json`: Extension configuration, permissions, content script declarations
- `extension/background.js`: Service worker entry; handles OAuth, email tracking, reply checking
- `extension/content.js`: Gmail page content script; detects Send, injects tracking pixel
- `server/index.js`: Express app entry; mounts all API routes, Prisma client initialization
- `web/src/main.jsx`: React root; mounts App component to DOM

**Configuration:**

- `server/.env`: (gitignored) Server environment: `REACH_SECRET`, `GEMINI_KEY`
- `server/.env.example`: Template for server env vars
- `web/.env.local`: (gitignored) Web environment: `VITE_GEMINI_API_KEY`
- `extension/config.js`: (gitignored) Extension config template; runtime API URLs
- `web/vite.config.js`: Vite build config; React plugin, Tailwind CSS
- `server/package.json`: Server deps (Express, Prisma, Zod)
- `web/package.json`: Web deps (React, dnd-kit, Tailwind CSS)

**Core Logic:**

**Extension:**
- `extension/classifier.js`: Keyword-based cold outreach detection; company extraction heuristics
- `extension/background.js`: Gmail API integration; trackLatestSent, checkReplies, message routing
- `extension/content.js`: Compose window detection; tracking pixel injection; Send button interception
- `extension/sidebar.js`: Gmail sidebar UI for recent tracked emails, stats widget

**Server:**
- `server/index.js`: All REST endpoints (outreach CRUD, tracking pixel, insights, email drafting)
- `server/emailFinder.js`: Email discovery utility; uses company/name/domain to find contact email
- `server/prisma/schema.prisma`: Data models (Outreach, TrackingPixel, OpenEvent)

**Web:**
- `web/src/App.jsx`: Main layout; tabs (Active/Archived/Insights), toolbar, kanban board or list view
- `web/src/hooks/useOutreach.js`: Custom hook managing record state; fetch, update, optimistic mutations
- `web/src/components/KanbanBoard.jsx`: Drag-drop board using dnd-kit; column-based layout
- `web/src/components/Sidebar.jsx`: Detail panel; notes, next action date, conversation thread, AI feedback
- `web/src/lib/api.js`: Fetch wrappers for server API calls
- `web/src/lib/gemini.js`: Gemini API integration for email drafting
- `web/src/lib/utils.js`: Date formatting, color utilities

**Testing:**
- Not found. No test files in codebase.

## Naming Conventions

**Files:**

- **camelCase for JS/JSX**: `background.js`, `useOutreach.js`, `OutreachCard.jsx`
- **PascalCase for React components**: `Sidebar.jsx`, `KanbanBoard.jsx`, `InsightsPanel.jsx`
- **kebab-case for icon components**: `HeartIcon.jsx`, `ChatIcon.jsx` (descriptive suffixes)
- **No file extension for utilities**: e.g. `schema.prisma`, `manifest.json`

**Directories:**

- **Plural for feature groupings**: `components/`, `icons/` (within components)
- **Singular for utilities**: `lib/`, `hooks/`
- **Feature-based**: `prisma/migrations` for database schema history

## Where to Add New Code

**New Feature:**
- **Backend endpoint**: Add to `server/index.js` route handlers
- **Database schema change**: Modify `server/prisma/schema.prisma`; run `npm run db:migrate`
- **Frontend component**: Create in `web/src/components/` (PascalCase.jsx)
- **Hook for state**: Add to `web/src/hooks/`
- **Server utility**: Add to `server/emailFinder.js` or new `server/[feature].js` file

**New Component/Module:**
- **React Component**: `web/src/components/[FeatureName].jsx` with functional component syntax
- **Extension Script**: `extension/[feature].js` as ES module (import/export)
- **Server Module**: `server/[feature].js` as ES module
- **Shared Constant**: Add to `shared/constants.js` with named export

**Utilities:**
- **API helpers**: `web/src/lib/api.js` (fetch wrappers for server endpoints)
- **Date/string utilities**: `web/src/lib/utils.js`
- **Classification logic**: `extension/classifier.js`

## Environment Variables

**Server (`.env`):**
- `REACH_SECRET`: Optional secret header for `/api/find-email` and `/api/draft-email` endpoints
- `GEMINI_KEY`: Google Generative AI API key for email drafting
- `DATABASE_URL`: Prisma datasource URL (default: `file:../dev.db`)

**Web (`.env.local`):**
- `VITE_GEMINI_API_KEY`: Client-side Gemini API key (embedded in frontend)

**Extension (`config.js.template` → `config.js`):**
- `serverApiBase`: URL of Express API server (default: `http://localhost:3001/api`)
- `dashboardUrl`: URL of web dashboard (default: `http://localhost:5173`)
