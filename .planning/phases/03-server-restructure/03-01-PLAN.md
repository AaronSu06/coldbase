Create server/routes/outreach.js using express.Router(). Import prisma from '../lib/prisma.js' and z from 'zod'.

Define two Zod schemas inline directly above the POST handler:

```javascript
const CreateOutreachSchema = z.object({
  threadId:     z.string().min(1),
  company:      z.string().min(1),
  contactEmail: z.string().email(),
}).passthrough();

const PatchOutreachSchema = CreateOutreachSchema.partial();
```

Implement 4 route handlers on the router (NOT on app):

GET /: Pagination with limit/offset. Parse query params: `limit = Math.min(parseInt(req.query.limit) || 100, 500)` and `offset = parseInt(req.query.offset) || 0`. Run `prisma.outreach.findMany({ orderBy: { sentDate: 'desc' }, take: limit, skip: offset })` and `prisma.outreach.count()` in parallel via `Promise.all`. Return `res.json({ data: records, total })`. Wrap in try/catch calling `next(e)`.

POST /: Run `CreateOutreachSchema.safeParse(req.body)`. If not `.success`, return `res.status(400).json({ error: 'Validation Error', message: parsed.error.issues.map(i => i.message).join('; '), statusCode: 400 })`. On success, `prisma.outreach.create({ data: parsed.data })`, return 201 with the record. On error, call `next(e)` — NOT res.status(500). The global error handler in index.js will handle P2002 (409) and P2025 (404).

PATCH /:threadId: Run `PatchOutreachSchema.safeParse(req.body)`. Same 400 pattern on failure. On success, `prisma.outreach.update({ where: { threadId: req.params.threadId }, data: parsed.data })`, return the record. On error, call `next(e)`.

DELETE /:threadId: `prisma.outreach.delete({ where: { threadId: req.params.threadId } })`, return 204. On error, call `next(e)`.

Export the router as default: `export default router;`

Critical: Express 4 does NOT auto-forward async errors. Every async handler MUST have explicit try/catch with next(e) in the catch — never use throw without catch.
