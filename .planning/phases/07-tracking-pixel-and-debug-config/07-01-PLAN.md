<task type="auto">
  <name>Task 2: Pre-fetch serverBase in tracking.js init, use in injectTrackingPixel</name>
  <files>extension/tracking.js</files>
  <action>
tracking.js is a classic script — no ES module imports allowed. Use chrome.runtime.sendMessage to pre-fetch config.

1. Add a module-local variable at the top of the IIFE (after the existing `let _state = null` block):
```js
// Cached server base URL from config — pre-fetched in init()
let _serverBase = 'http://localhost:3001';
```

2. Modify `injectTrackingPixel` to use `_serverBase` instead of the hardcoded string:
```js
function injectTrackingPixel(editorEl, trackingId) {
  const img = document.createElement('img');
  img.src = _serverBase + '/track/' + trackingId + '.gif';
  img.width = 1; img.height = 1;
  img.style.cssText = 'display:block;width:1px;height:1px;opacity:0;position:absolute;pointer-events:none;';
  img.alt = '';
  editorEl.appendChild(img);
}
```

3. Modify `init(state)` to pre-fetch the config via sendMessage before caching state. The fetch is async but init stores the result when it resolves — subsequent pixel injections use the cached value. If the fetch fails or the extension context is gone, the fallback `'http://localhost:3001'` remains in `_serverBase`.

Replace:
```js
function init(state) {
  _state = state;
  log.debug('ReachTracking initialized.');
}
```

With:
```js
function init(state) {
  _state = state;
  log.debug('ReachTracking initialized.');
  chrome.runtime.sendMessage({ type: 'GET_RUNTIME_CONFIG' }, (resp) => {
    if (chrome.runtime.lastError) {
      log.warn('GET_RUNTIME_CONFIG failed — using fallback serverBase:', chrome.runtime.lastError.message);
      return;
    }
    if (resp?.ok && resp.config?.serverBase) {
      _serverBase = resp.config.serverBase;
      log.debug('serverBase loaded from config:', _serverBase);
    }
  });
}
```

Do NOT add ES module imports. Do NOT modify any other functions.
