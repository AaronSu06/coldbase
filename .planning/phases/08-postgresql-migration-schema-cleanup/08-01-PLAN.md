Replace the $queryRaw template literal with the PostgreSQL-compatible version. Two constructs must be fixed, plus quoted identifiers added:

Replace the entire $queryRaw block (lines 15-24) with:
```javascript
const rows = await prisma.$queryRaw`
  SELECT
    EXTRACT(HOUR FROM "sentDate")::INTEGER AS hour,
    COUNT(*) AS sent_count,
    SUM(CASE WHEN "repliedAt" IS NOT NULL THEN 1 ELSE 0 END)::INTEGER AS replied_count
  FROM "Outreach"
  WHERE archived = false
  GROUP BY hour
  ORDER BY hour
`;
```

Changes:
1. strftime('%H', sentDate) → EXTRACT(HOUR FROM "sentDate")::INTEGER (PostgreSQL date function)
2. archived = 0 → archived = false (PostgreSQL native boolean, not SQLite integer)
3. Outreach → "Outreach" (PostgreSQL case-sensitive quoted identifier; Prisma maps model names this way)
4. repliedAt → "repliedAt", sentDate → "sentDate" (camelCase columns require quoting in PostgreSQL raw SQL)
5. SUM result cast to ::INTEGER (PostgreSQL SUM returns bigint; explicit cast is cleaner than Number() in JS)

Do not change the rows.map() block below — Number() casts still work fine.
