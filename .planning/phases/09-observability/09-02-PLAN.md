  <behavior>
    - A GET request to any route produces a console.log call with valid JSON
    - The JSON contains: type="request", timestamp (ISO string), method, path (no query string), status (final status code), durationMs (number >= 0), query (object)
    - The JSON does NOT contain the value of the x-reach-secret header
    - The log fires AFTER the response is committed (res.on('finish')) so status is the final committed value
    - A 401 response logs status 401 (not 200)
  </behavior>
  <action>
    Create `server/middleware/requestLogger.js`:

    ```javascript
    export default function requestLogger(req, res, next) {
      const start = Date.now();
      res.on('finish', () => {
        console.log(JSON.stringify({
          type: 'request',
          timestamp: new Date().toISOString(),
          method: req.method,
          path: req.path,
          status: res.statusCode,
          durationMs: Date.now() - start,
          query: req.query,
        }));
        // x-reach-secret intentionally omitted — never log auth credentials
      });
      next();
    }
    ```

    Key decisions (per CONTEXT.md):
    - `req.path` not `req.url` — path only, no query string duplication
    - `res.statusCode` inside 'finish' callback only — never read early
    - `type: 'request'` included (Claude's discretion) — discriminator for Phase 10 error log lines
    - `timestamp` included (Claude's discretion) — structured logs without timestamps lose correlation value
    - No try/catch — JSON.stringify on these primitive values cannot throw
    - No IP address — PII, no debugging value for single-user app (per CONTEXT.md)
  </action>
