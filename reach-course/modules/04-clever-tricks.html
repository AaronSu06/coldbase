Trick 1: Fuzzy keyword matching

The classifier doesn't just look for exact words — it uses edit distance to handle typos and spelling variations. So "internships" and "internhsips" both match the "intern" keyword group.

CODE 

function tokenMatchesVariant(token, variant) {
  if (token === variant) return true;
  const limit = variant.length >= 9 ? 2
    : variant.length >= 5 ? 1
    : 0;
  return editDistanceWithinLimit(token, variant, limit);
}
PLAIN ENGLISH

Check if a word from the email matches a keyword variant.

If it's an exact match, immediately return true — no need to do math.

Set a tolerance for typos based on the variant's length:

Long words (9+ chars) allow 2 typos. Medium words (5-8 chars) allow 1. Short words allow none.

Check if the word is close enough using the edit distance algorithm.

The classifier groups keywords by concept — not individual words. So "intern," "interns," "internship," and "interning" all count as ONE match. This prevents an email stuffed with synonyms from scoring artificially high.

Trick 2: Company name extraction via Clearbit

Once Reach knows it's a cold email, it needs the company name. It tries four strategies in order — from fastest to most expensive — and stops as soon as one works. Like a detective working through a list of leads.

1
[Company Name] in brackets

Subject line pattern: "[Anthropic] Software Engineer Role" — extracts instantly, no API call needed.

2
Company — prefix

Subject line pattern: "OpenAI — Interested in joining?" — fast regex match.

3
— Company suffix

Subject line pattern: "Software Engineering — Google" — fast regex match in reverse.

4
Clearbit API lookup

Scans the email body for capitalized word sequences, scores them by frequency, then validates the top candidates against Clearbit. Slowest but catches any company name.

Trick 3: The invisible tracking pixel

How does Reach know when someone opens your email? It uses an email tracking pixel — the same technique used by major email marketing platforms. When the server saves an outreach record, it generates a unique tracking ID and embeds an invisible image URL in the email.

Limitation to know

Many email clients (especially Gmail's web interface) now proxy images through their own servers to block tracking. An "open" event may mean Gmail pre-loaded the images — not that a human actually read the email. Open rate data should be treated as a signal, not ground truth.

Trick 4: The quota gate (free vs pro)

Reach has a freemium model — free users get a limited number of email lookups per month, pro users get more. This is enforced by a middleware function called checkQuota that sits in front of the expensive API endpoints.

CODE 

// server/app.js — rate limiters stack up
app.use('/api/find-email',
  expensiveRateLimit,
  checkQuota
);
app.use('/api/suggest-domains',
  expensiveRateLimit
);
app.use('/api/draft-email',
  expensiveRateLimit
);
PLAIN ENGLISH

For the email-lookup endpoint — the most expensive one — apply TWO guards:

First, rate limiting: max 10 requests per 15 minutes per IP address.

Then, quota check: has this user hit their monthly lookup limit?

Domain suggestions: just rate limited (no quota — less expensive).

AI email drafting: just rate limited (no quota — Gemini is cheap at this scale).

💡
Key Insight: Layered defenses

Notice how Reach stacks two protections: a rate limiter (protects against bursts) and a quota gate (enforces monthly limits). These solve different problems — rate limiting handles abuse in a short window, quotas handle cumulative costs over time. When you ask AI to "add protection" to an endpoint, clarify which type you need.

Trick 5: The reply-check alarm

Reach can't stay connected to Gmail 24/7 — that would drain your battery and violate Chrome's service worker rules. Instead, it uses Chrome's alarm system: a built-in scheduler that fires a named event every N minutes, even when Chrome is in the background.

CODE 

chrome.runtime.onInstalled.addListener(() => {
  chrome.alarms.create('coldbase-reply-check', {
    periodInMinutes: 30
  });
});

chrome.alarms.onAlarm.addListener(async (alarm) => {
  if (alarm.name === 'coldbase-reply-check') {
    const token = await getAuthToken(false);
    await checkReplies(token);
  }
});
PLAIN ENGLISH

When the extension is first installed, set up a repeating alarm...

Give it a name ("coldbase-reply-check") so we can identify it later.

Set it to fire every 30 minutes — automatically, forever.

Whenever any alarm fires, check if it's our reply-check alarm.

If so, get the Gmail access token (silently, without showing a login popup).

Call checkReplies, which scans Gmail for replies to tracked emails.
