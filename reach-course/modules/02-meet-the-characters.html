Meet the Characters

The five actors that power Reach — and exactly what each one is responsible for

Five specialists, one team

Reach is not one program — it's a team of five specialists. Each one has a specific job and only talks to the others when it needs to. Think of it like a newsroom: reporters, editors, printers, and distributors — each expert at one thing, working in sync.

📝
content.js — The Gmail Observer

Injected directly into Gmail. Watches for compose windows, detects sends, and injects the Reach sidebar widget. It's the only part of the app that can “see” what's on your screen.

⚡
background.js — The Service Worker

Runs silently in the background 24/7. Calls the Gmail API, runs the reply-check alarm every 30 minutes, and routes messages between the extension and the server.

🏗️
server/app.js — The Express Server

A Node.js API server that handles all the business logic. Saves outreach records, checks quotas, generates AI drafts, and manages user accounts.

🗄️
PostgreSQL Database — The Filing Cabinet

Stores everything permanently: users, outreach records, tracking pixels, and open events. Managed through Prisma, which translates JavaScript into database language.

🖥️
web/src — The React Dashboard

The kanban board you interact with at coldbase.live. Built with React — fetches data from the server and renders your outreach cards with drag-and-drop.

Where each character lives in the code

When you ask an AI to “add a feature to the sidebar,” knowing which folder to point it at is the difference between a clean change and a mess. Here's the map:

extension/ Everything that runs inside Chrome
content.js Gmail observer — injects UI, detects sends
background.js Background service worker — Gmail API calls, alarms
classifier.js Is this a cold email? Keyword matching + fuzzy logic
compose-widget.js The sidebar panel that appears when you compose
manifest.json Extension ID card — tells Chrome what permissions it needs
server/ The backend — saves data, serves the API
app.js Main server — wires up all routes and middleware
routes/ Handlers for each API endpoint (outreach, auth, billing…)
prisma/ Database schema — defines what data looks like
web/src/ The React dashboard
App.jsx Main app shell — routing, state, toolbar
components/ KanbanBoard, OutreachCard, Sidebar, TopNav…
hooks/ useAuth, useOutreach — shared data-fetching logic

What gets saved — the data model

The database has four tables. Think of each one as a sheet in a spreadsheet — every row is one record, every column is a field.

👤

User

Email, password hash, Google ID, subscription plan (free/pro), Stripe IDs, quota usage. One row per person.

📨

Outreach

Company, contact name, email, subject, sent date, status (Sent / Replied / Interviewing / Offer / Ghosted), reply info, notes. One row per cold email sent.

🔍

TrackingPixel

Links each sent email to a unique tracking ID. When a recipient opens the email, their mail client loads an invisible image that records the event.

👁️

OpenEvent

Records every time the tracking pixel fires — timestamp, user agent, IP address. Many open events can belong to one TrackingPixel.

💡
Key Insight: Relationships between tables

One User has many Outreaches. One Outreach has one TrackingPixel. One TrackingPixel has many OpenEvents. This “one-to-many” relationship is the fundamental pattern in almost every database ever built.

The team chat — components in conversation

Here's what the conversation looks like when you manually click “Refresh” on the dashboard to re-scan your Gmail:

📊
Dashboard (React)

User clicked Refresh. I need to trigger a rescan AND a reply check. Let me post a message to the window.

🔌
dashboard-sync.js (Relay)

I see that window.postMessage. I'll forward it into the extension as a chrome.runtime message.

⚡
background.js

Got it! Calling Gmail API now... found 3 new messages. Sending them to the server.

🏗️
Server (Express)

Received 3 outreach records. Saving to database. Done — 200 OK!

🔌
dashboard-sync.js (Relay)

Operation complete! Posting the result back to the window with the same requestId so Dashboard knows to stop spinning.

📊
Dashboard (React)

Got my response! Calling refresh() to re-fetch the outreach list. Board is up to date.
