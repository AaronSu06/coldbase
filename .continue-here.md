<decisions_made>

- **JWT (not sessions)** ‚Äî chose stateless JWT so Chrome/Firefox/Safari extensions can all use the same token without server-side session management
- **7-day expiry, no refresh tokens** ‚Äî simple for now; refresh tokens can be added later
- **No email verification** ‚Äî skipped for now; can add later when email sending service is set up
- **`bcrypt` cost factor via env var** ‚Äî `BCRYPT_ROUNDS` defaults to 12, set to 1 in tests for speed
- **Auth routes at `/api/auth` before `requireAuth` at `/api`** ‚Äî Express order-of-registration ensures signup/login are public
- **`findFirst({ where: { threadId, userId } })` instead of `findUnique`** ‚Äî scopes single-record lookups to the authenticated user
- **PATCH strips `userId`/`id`/system fields** ‚Äî closes ownership reassignment vulnerability from `.passthrough()` Zod schema
- **Extension token sync via content script** ‚Äî dashboard-sync.js runs on dashboard URL, reads `reach_token` from localStorage, messages background.js which calls `setReachToken()`

</decisions_made>
